# Сценарий демонстрации на защите

**Длительность:** 3–5 минут  
**Предварительно:** `make up`, `car:parser`, создан администратор, есть 1–2 клиента и автомобили IN_STOCK

---

## 1. Публичная витрина (45 с)

| Действие | Что показать комиссии |
|----------|----------------------|
| Открыть http://localhost | Каталог DriveLine |
| Вкладка «Новый» / «Б/У» | Фильтрация по типу |
| Указать на карточку | Цена, фото, только авто в наличии |

**Фраза:** «Посетитель без авторизации видит актуальный склад; данные кэшируются через CarService».

---

## 2. Вход в админ-панель (30 с)

| Действие | Что показать |
|----------|--------------|
| `/admin` | Форма входа MoonShine |
| Вход под admin | Боковое меню: Система, Автомобили, Сделки, Клиенты |

**Фраза:** «Бэк-офис защищён сессией; права настраиваются по ролям и матрице permissions».

---

## 3. Сделка продажи (90 с)

| Шаг | Действие | Ожидаемый эффект |
|-----|----------|------------------|
| 1 | Сделки → Продажа → Создать | Форма сделки |
| 2 | Выбрать клиента и авто IN_STOCK | — |
| 3 | Статус «Подготовка», сохранить | В списке cars — «Резерв» |
| 4 | Обновить витрину (F5) | Авто может исчезнуть из каталога |
| 5 | Статус «Завершена» | cars → «Продан» |
| 6 | (опционально) «Отменена» | Возврат «В наличии» |

**Фраза:** «Бизнес-правило реализовано в SailService::syncSellCarStatus — без ручного изменения склада».

---

## 4. Отчёт (60 с)

| Шаг | Действие |
|-----|----------|
| 1 | Система → Отчёты → Создать |
| 2 | Период, тип «Продажа» |
| 3 | Убедиться, что работает очередь (`make queue` в отдельном терминале) |
| 4 | Обновить запись — статус «Завершён», скачать XLSX |

**Фраза:** «Тяжёлая операция вынесена в GenerateReportJob и Redis».

---

## 5. Резерв: если демо падает

- Показать скриншоты из `appendices/appendix-d-screenshots.md`
- Открыть приложение Ж — листинг `syncSellCarStatus`
- Показать успешный GitHub Actions (CI)

---

## Типовые вопросы и ответы

| Вопрос | Ответ |
|--------|-------|
| Почему Laravel? | ORM, миграции, очереди, экосистема, скорость разработки |
| Почему MoonShine? | Готовый CRUD и авторизация для бэк-офиса |
| Как обеспечена целостность? | FK в БД + транзакции в SailService + enum-статусы |
| Чем отличается от 1С? | Учебный full-stack проект, открытый код, кастомная логика сделок |
| Нормальные формы? | 3НФ, N:M через option_sale, права в отдельной таблице |
