# Заключение

В ходе выполнения курсовой работы по дисциплине «Проектирование информационных систем» спроектирована и реализована информационная система **DriveLine** для автоматизации учёта автомобилей, клиентов и сделок автосалона.

В **аналитической части** рассмотрена предметная область автосалона, построена функциональная модель в нотации IDEF0, обоснована необходимость применения вычислительной техники, сформулирована постановка задачи и проведён сравнительный анализ существующих программных решений.

В **проектной части** разработаны диаграммы потоков данных, концептуальная, логическая и физическая модели базы данных в третьей нормальной форме, а также сценарии диалога для административной панели MoonShine и публичной витрины.

В **части реализации** создано веб-приложение на стеке Laravel 13, PostgreSQL, MoonShine и Redis с разделением на публичный каталог и закрытый бэк-офис, сервисным слоем бизнес-логики и фоновой генерацией отчётов. Ключевое бизнес-правило — автоматическая синхронизация статуса автомобиля при изменении статуса сделки продажи — реализовано в модуле `SailService`.

**Практическая ценность:** снижение риска ошибок складского учёта, ускорение подготовки отчётности, централизованное хранение данных о клиентах и сделках, разграничение прав доступа сотрудников.

**Перспективы развития:** REST API для мобильного приложения; онлайн-заявка с витрины; интеграция с банковскими API по кредитам; расширение автотестов бизнес-логики; доработка аналитической панели для руководителя.
